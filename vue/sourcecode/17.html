# 响应式属性的增加和删除

# set

向响应式对象添加一个新的属性，并确保这个新属性同样是响应式的，并且能触发视图更新。

参考

# 使用

两种方式

  • 全局API :在src/core/global-api/index.js中注册全局set方法Vue.set=set
  • $set: 在src/core/instance/state.js中定义$set: Vue.prototype.$set = set

参数

  • {Object | Array} target
  • {string | number} propertyName/index
  • {any} value

返回值

设置的值

WARNING

给响应式数据添加新的属性,不能给实例或者实例data选项添加响应式属性。

不能给原始类型的值添加响应式属性

# 源码分析

set

export function set (target: Array<any> | Object, key: any, val: any): any {
  if (process.env.NODE_ENV !== 'production' &&
    (isUndef(target) || isPrimitive(target))
  ) {
    warn(`Cannot set reactive property on undefined, null, or primitive value: ${(target: any)}`)
  }
  if (Array.isArray(target) && isValidArrayIndex(key)) {
    target.length = Math.max(target.length, key)
    target.splice(key, 1, val)
    return val
  }
  if (key in target && !(key in Object.prototype)) {
    target[key] = val
    return val
  }
  const ob = (target: any).__ob__
  if (target._isVue || (ob && ob.vmCount)) {
    process.env.NODE_ENV !== 'production' && warn(
      'Avoid adding reactive properties to a Vue instance or its root $data ' +
      'at runtime - declare it upfront in the data option.'
    )
    return val
  }
  if (!ob) {
    target[key] = val
    return val
  }
  defineReactive(ob.value, key, val)
  ob.dep.notify()
  return val
}

通过流程图分析data初始化过程

从流程图可以看到由于{age: 30}是一个对象，所以observe({age: 30})返回值为Oberve实例并且具有dep属性。在render访问数据tom时,会触发getter，这个过程中会执行childOb.dep.depend收集依赖。

因此通过set方法this.$set(this.tom, 'sex', 'male'),会拿到ob=this.tom.__ob__,然后手动调用notify方法通知更新。

从上述分析可以看到，响应式属性getter中的childOb.dep.depend()其实是为了set方法定制，在render访问响应式数据执行getter过程中收集依赖，在执行set方法时执行notify方法才能准确通知响应的watcher更新。

值得注意的是

1） 执行set添加响应式属页面首次更新是在set方法中手动调用notify方法

2） 由于通过set方法给添加的属性变成响应式属性，在后续该属性发生变化时，会执行setter，setter中会执行dep.notify。

通过protoAugment方法将数组的原型指向arrayMethods.

在src/core/instance/oberver/array.js中，对arrayMethods进行代理。

const methodsToPatch = [
  'push',
  'pop',
  'shift',
  'unshift',
  'splice',
  'sort',
  'reverse'
]
def(arrayMethods, method, function mutator (...args) {
    const result = original.apply(this, args)
    const ob = this.__ob__
    let inserted
    switch (method) {
      case 'push':
      case 'unshift':
        inserted = args
        break
      case 'splice':
        inserted = args.slice(2)
        break
    }
    if (inserted) ob.observeArray(inserted)
    // notify change
    ob.dep.notify()
    return result
  })

当数组操作方法为push,unshift或splice时，获取增加的元素（splice方法可能会给数组增加元素），然后调用observeArray方法，将增加的每一项变成响应式属性。并返回result，调用ob.dep.notify方法通知更新。

这里vue做了细节优化，没有observeArray（value），而是observeArray(inserted)

vue对常见的数组操作方法进行改写，改写的目的是在执行相关数组操作方法时会执行notify方法，派发更新。

  • 删除某个响应式属性需要通过Vue.delete或者$delete进行删除
  • 不能直接调用delete方法删除某个响应式属性,能删除响应式数据，但无法更新。

TIP

通过$set或$delete增删属性时，会调用ob.dep.notify派发更新。

删除某个响应式属性可以通过delete直接删除，但需要调用this.$forceUpdate方法强制更新，但只能通知调用forceUpdate方法的实例进行更新，但一个响应式属性可能会被不同组件所使用，因此直接delete配合forceUpdate方法有局限性，并不会通知每个渲染watcher作更新。
