# 计算属性和侦听属性

vue版本： 2.5.17-beta.0

# 计算属性

computed对象内的每个属性为计算属性，所谓计算属性，顾名思义也就是计算属性的值依赖某个响应式属性的值计算得到。 计算属性相对于直接在模板中使用表达式进行计算或者使用methods中的方法对响应式属性进行计算，优点是计算属性的值会被缓存，当计算属性依赖发生变化时才会重新进行计算。

# 计算属性的使用

new Vue({
	data() {
		return {
			msg: 'hello world'
		}
	},
	computed: {
		reverseMsg() {
			return this.msg.split('').reverse().join('')
		}
	}
})

defineComputed

export function defineComputed (
  target: any,
  key: string,
  userDef: Object | Function
) {
  const shouldCache = !isServerRendering()
  if (typeof userDef === 'function') {
    sharedPropertyDefinition.get = shouldCache
      ? createComputedGetter(key)
      : createGetterInvoker(userDef)
    sharedPropertyDefinition.set = noop
  } else {
    sharedPropertyDefinition.get = userDef.get
      ? shouldCache && userDef.cache !== false
        ? createComputedGetter(key)
        : createGetterInvoker(userDef.get)
      : noop
    sharedPropertyDefinition.set = userDef.set || noop
  }
  if (process.env.NODE_ENV !== 'production' &&
      sharedPropertyDefinition.set === noop) {
    sharedPropertyDefinition.set = function () {
      warn(
        `Computed property "${key}" was assigned to but it has no setter.`,
        this
      )
    }
  }
  Object.defineProperty(target, key, sharedPropertyDefinition)
}

进入App组件的实例化，在initState方法中会执行initComputed

const computedWatcherOptions = { computed: true }
function initComputed (vm: Component, computed: Object) {
  // $flow-disable-line
  const watchers = vm._computedWatchers = Object.create(null)
  // computed properties are just getters during SSR
  const isSSR = isServerRendering()

  for (const key in computed) {
    const userDef = computed[key]
    const getter = typeof userDef === 'function' ? userDef : userDef.get
    if (process.env.NODE_ENV !== 'production' && getter == null) {
      warn(
        `Getter is missing for computed property "${key}".`,
        vm
      )
    }

    if (!isSSR) {
      // create internal watcher for the computed property.
      watchers[key] = new Watcher(
        vm,
        getter || noop,
        noop,
        computedWatcherOptions
      )
    }

    // component-defined computed properties are already defined on the
    // component prototype. We only need to define computed properties defined
    // at instantiation here.
    if (!(key in vm)) {
      defineComputed(vm, key, userDef)
    } else if (process.env.NODE_ENV !== 'production') {
      if (key in vm.$data) {
        warn(`The computed property "${key}" is already defined in data.`, vm)
      } else if (vm.$options.props && key in vm.$options.props) {
        warn(`The computed property "${key}" is already defined as a prop.`, vm)
      }
    }1
  }
}

  getAndInvoke (cb: Function) {
    const value = this.get()
    if (
      value !== this.value ||
      // Deep watchers and watchers on Object/Arrays should fire even
      // when the value is the same, because the value may
      // have mutated.
      isObject(value) ||
      this.deep
    ) {
      // set new value
      const oldValue = this.value
      this.value = value
      this.dirty = false
      if (this.user) {
        try {
          cb.call(this.vm, value, oldValue)
        } catch (e) {
          handleError(e, this.vm, `callback for watcher "${this.expression}"`)
        }
      } else {
        cb.call(this.vm, value, oldValue)
      }
    }
  }

# 侦听属性源码分析

initWatch

initState方法中会执行initWatch

function initWatch (vm: Component, watch: Object) {
  for (const key in watch) {
    const handler = watch[key]
    if (Array.isArray(handler)) {
      for (let i = 0; i < handler.length; i++) {
        createWatcher(vm, key, handler[i])
      }
    } else {
      createWatcher(vm, key, handler)
    }
  }
}

遍历watch的每个属性得到handler,handler可以为数组(一般是单个函数)。然后调用createWatch为每个handler实例化watcher

function createWatcher (
  vm: Component,
  expOrFn: string | Function,
  handler: any,
  options?: Object
) {
  if (isPlainObject(handler)) {
    options = handler
    handler = handler.handler
  }
  if (typeof handler === 'string') {
    handler = vm[handler]
  }
  return vm.$watch(expOrFn, handler, options)
}

在stateMixin中定义了$watch方法

通过实例化 watcher 的方式，一旦我们 watch 的数据发送变化，它最终会执行 watcher 的 run 方法，执行回调函数 cb，并且如果我们设置了 immediate 为 true，则直接会执行回调函数 cb。最后返回了一个 unwatchFn 方法，它会调用 teardown 方法去移除这个 watcher。

  • 实例化watcher
  • 返回值为unwatchFn：() => watcher.teardown()
Vue.prototype.$watch = function (
    expOrFn: string | Function,
    cb: any,
    options?: Object
  ): Function {
    const vm: Component = this
    if (isPlainObject(cb)) {
      return createWatcher(vm, expOrFn, cb, options)
    }
    options = options || {}
    options.user = true
    const watcher = new Watcher(vm, expOrFn, cb, options)
    if (options.immediate) {
      cb.call(vm, watcher.value)
    }
    return function unwatchFn () {
      watcher.teardown()
    }
  }

# watcher.options

  • user
  • deep
  • sync
  • computed

在实例化watcher时options会有以下属性，其中调用$watch API时,user为true,计算属性实例化watcher时,computed为true。而deep或者sync在定义侦听属性时会用到。

deep

示例代码中对name进行观测时,deep为true，因此当age发生改变时，会执行回调函数。

在watcher的get方法中

get () {
    pushTarget(this)
    let value
    const vm = this.vm
    try {
      value = this.getter.call(vm, vm)
    } catch (e) {
      if (this.user) {
        handleError(e, vm, `getter for watcher "${this.expression}"`)
      } else {
        throw e
      }
    } finally {
      // "touch" every property so they are all tracked as
      // dependencies for deep watching
      if (this.deep) {
        traverse(value)
      }
      popTarget()
      this.cleanupDeps()
    }
    return value
  }

在实例化userWatcher时，会执行get方法,计算的value值即为data中定义的属性student,由于deep为true,会执行traverse(student)方法。

traverse方法本质是遍历student的每个属性,从而触发响应式属性的getter，从而响应式数据会将当前的userWacther作为订阅者添加到dep.subs中。此时无论响应式属性层级有多深，设置了deep，无论哪一层级的数据发生变化都会调用dep.notify方法，通知subs中的订阅者watcher ，最终（有的在下一个tick）会执行watcher.run方法，从而执行回调函数。

上面分析了deep的作用,deep为对象类型的响应式属性监听提供了解决方案，原理是遍历触发属性getter收集依赖，这样当属性发生变化时会执行回调函数。

sync

当watcher的options的sync属性设置为true时，数据变化引发的回调函数的执行不会放到下一个tick中，而是立即执行watcher.run方法,从而立即执行回调函数。

update () {
  if (this.computed) {
    // ...
  } else if (this.sync) {
    this.run()
  } else {
    queueWatcher(this)
  }
}

# 计算属性和侦听属性的区别

相同点

  • 计算属性和侦听属性都会示例化一个watcher

不同点

  • 计算属性在computed中定义，每个计算属性都会被示例化一个watcher,其中options.computed为true
  • 侦听属性在watch对象中定义，或者通过$watch方法定义侦听属性。
  • 计算属性一般是依赖某个响应式数据，而侦听属性侦听的目标可以是data中的某个响应式属性，也可以是computed中某个计算属性
  • 计算属性在使用时的场景主要是根据某个响应式数据进行计算，计算的值在模板中会用到，对于依赖没有发生变化时，由于标志符dirty为false，因此直接返回结果，而不会重新计算。这是一种优化。
  • 侦听属性一般用于在值发生变化时，进行相关的业务处理，比如发送请求等。
