# entry

entry打包入口，目的是告诉webpack从哪个文件开始构建文件依赖。默认值为./src/index.js

# context

context 可以理解为资源入口的路径前缀，在配置时要求使用绝对路径，context可以省略，默认值为当前工程的根目录

当打包多入口文件时，借助context可以使entry书写更加简洁

// webpack.config.js

module.exports = {
    context: path.resolve(__dirname, './src'),
    entry: {
        foo: './foo.js',
        bar: './bar.js'
    }
}

TIP

当我们在开发多页应用时，如果多页入口业务代码合并后的文件体积满足要求，可以考虑使用数组形式，这样可以利用缓存机制提高页面访问速度，这也是性能优化的一点。

# 函数类型

可以动态获取入口

webpack.config.js

    module.exports = {
        entry: () => './src/index.js'
    }
    // or promise
    module.exports = {
        entry: () => new Promise(res => {
            setTimeout(() => {
                resolve({
                    app: './src/app.js',
                    lib: './src/lib.js'
                })
            })
        })
    }

# 分离app和vendor

TIP

在v4版本以前，使用CommonsChunkPlugin分离vendor和app模块，在v4版本使用optimization.splitChunks。

WARNING

提取公共代码一般常见于多页应用中，尽管如此，但在单页应用中我们一般会将业务代码和vendor代码进行分离。 而在多页应用中，可能有相关的业务代码会被不同入口所共用，这时我们可以配置相关规则，将这部分代码提取出来。
