# output

在配置文件中设置output选项，目的是为了让webpack将编译后的文件放到指定位置。如果不设置output，默认为dist目录

# filename

控制输出资源的文件名，类型为字符串，filename可以是一个包含路径的字符串，如果路径中的目录不存在，就会在输出资源时创建该目录。

module.exports = {
    entry: './src/app.js',
    output: {
        filename: './js/bundle.js'
    }
}

控制客户端缓存可采取[name].[chunkhash].js

# chunkFilename

指定非入口文件打包生成的文件名,默认为[id].js

webpack.config.js

module.exports = {
    entry: './src/index.js',
    output: {
        filename: '[name].bundle.js',
        chunkFilename: '[name].js'  //这里的name为从0到1自增的id
        path: path.resolve(__dirname, 'dist')
    }
}

# path

path指定资源的输出位置,绝对路径。

mudule.exports = {

    output: {
        filename: 'bundle.js',
        path: path.join(__dirname, 'dist')

# publicPath

publicPath指定资源的请求位置

WARNING

path: 输出位置,打包后的资源输出目录，一般指定为dist目录 publicPath: 由JS或CSS所请求的间接资源路径。页面中的资源分为两种，一种由html页面直接请求的，比如通过script标签加载的JS;另一种由JS或CSS请求的，比如异步加载的JS,从CSS请求的图片字体等。publicPath的作用就是指定这部分间接资源的请求位置。

publicPath的三种形式

  • 与HTML相关

将publicPath指定为HTML相对路径，在请求这些资源时会以当前页面HTML路径加上相对路径，构成实际请求的url

// html地址 https://demo.com/home/index.html
// 异步加载的资源名为 0.chunk.js
publicPath: '', //实际路径为https://demo.com/home/0.chunk.js
publicPath: './js', //实际路径为https://demo.com/home/js/0.chunk.js
publicPath: '../assets', //实际路径为https://demo.com/assets/0.chunk.js

WARNING

webpack的devServer也有publicPath属性，它的作用是指定webpack-dev-server的静态资源服务路径。

TIP

为了避免开发环境和生产环境产生不一致而造成的疑惑，我们可以将webpack-dev-server的publicPath与Webpack中的output.path保持一致，这样在任何环境下资源输出的目录都是相同的

# 小结

webpack的output属性有很多，这里介绍了常见的filename,path,chunkFilename,publicPath等，这四个能帮我们实现以下打包功能

  • 指定每个entry编译后生成的文件名
  • 指定打包后的资源输出位置
  • 指定生成的chunk的文件名
  • 指定静态资源中的间接资源请求地址。

在后面介绍使用webpack打包library时，会涉及到output的其他相关属性。
