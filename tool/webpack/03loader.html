# loader

# 为什么需要loader

webpack只能处理js类型文件，对于其他类型文件，需要相应的loader进行处理，如果不配置相关loader,webpack会无法处理相关类型文件，从而在打包的过程中抛出错误。

# 使用loader

在你的应用中有三种方式使用loader

  • 配置文件，在webpack.config.js中使用loader
  • 内联（Inline）形式使用loader
  • 在CLI中使用loader

TIP

由于很少在内联和CLI中使用loader，本文就不加以说明，想了解的可以去官网查看如何使用。

在webpack.config.js的module.rules中配置loader

module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        use: [
          // style-loader
          { loader: 'style-loader' },
          // css-loader
          {
            loader: 'css-loader',
            options: {
              modules: true
            }
          },
          // sass-loader
          { loader: 'sass-loader' }
        ]
      }
    ]
  }
};

# loader的特点

  • Loaders可以链式调用，每个loader对处理中的资源进行转换，当有多个loader时，loader的调用是逆序的。第一个loader将转变后的结果传递给下一个loader，最终webpack希望链式loader的最后一个loader返回的是javascript代码。
  • Loaders可以是同步或者异步的。
  • Loaders运行在Node.js环境，并且执行在Node.js中任何可能的任务。
  • 通过options对象可以配置loader（通过query参数设置options仍然支持，但是不建议。
  • 普通模块借助package.json文件中的main和loader字段可以导出loader。
  • Plugins可以赋予loader更多特性。
  • Loaders可以传递额外的文件。

WARNING

上面说到链式的loader的执行是逆序的，但是可以通过enforce字段改变loader的执行顺序

  • pre: 在所有正常loader之前执行,比如eslint-loader
  • post: 在所有loader之后执行

# 常见Loaders

# babel-loader

babel相关模块的作用

  • babel-loader: 是使Babel与Webpack协同工作的模块
  • @babel/core: Babel编译器的核心模块
  • @babel/preset-env: babel官方推荐的预置器，可根据用户设置的目标环境自动添加所需的插件和补丁来编译es6+代码。

babel其他注意事项

{
    test: /\.js$/,
    use: {
        loader: 'babel-loader',
        options: {
            // 启用缓存机制，加快打包速度
            cacheDirectory: true,
            presets: [
                ["env", {
                    modules: false  
                
                }]
            ]
        }，
        exclude: /node_modules/
    
    }
}

# vue-loader

对于.vue类型的文件需要配置vue-loader

npm install -D vue-loader vue-template-compiler
1

webpack.config.js

// webpack.config.js
const VueLoaderPlugin = require('vue-loader/lib/plugin')

module.exports = {
  module: {
    rules: [
      // ... other rules
      {
        test: /\.vue$/,
        loader: 'vue-loader'
      }
    ]
  },
  plugins: [
    // make sure to include the plugin!
    new VueLoaderPlugin()
  ]
}

# 编写loader

loader是一个函数，接受一个参数（源文件，或者被其他loader处理后的文件） loader应该是一个纯函数，对于相同的输入，需保证相同的输出，所以这里 使用this.cacheable开启缓存

if(this.cacheable) { this.cacheable() }
1

配置options

npm install loader-utils
1
const loaderUtils = require('loader-utils')
module.exports = function(content) {
    if(this.cacheable) {
        this.cacheable()
    }
    // 获取和打印options
    var options = loaderUtils.getOptions(this) || {}
    console.log(options)
    // 处理content
}
