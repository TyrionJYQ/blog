# 概览

webpack是一个现代的模块打包器，他能够处理前端开发中所涉及的各种类型的资源文件，按照默认或者指定的配置将资源进行打包优化。

TIP

webpack 自身只支持打包js类型文件，对于非js类型文件需要借助loader进行处理。

webpack 功能强大，但使用和掌握是一个需要积累的过程，学习webpack首先需要弄清四大核心概念

  • entry
  • output
  • loader
  • plugin

# entry

entry指定打包入口，告诉webpack 从哪个文件开始进行打包，构建打包中的依赖，最终输出打包后的文件

# output

output 是webpack打包后的文件存放位置，一般为dist目录，或者为build目录

# loader

loader在module对象中指定，一般格式如下

module: [
  	rules: [
  		{
  			test,
  			use
		}
  	]
]

# plugins

webpack中的插件本质是一个具有apply方法的js对象。apply方法会被webpack编译器调用，能够访问webpack整个编译周期。

const pluginName = 'ConsoleLogOnBuildWebpackPlugin';

class ConsoleLogOnBuildWebpackPlugin {
  apply(compiler) {
    compiler.hooks.run.tap(pluginName, compilation => {
      console.log('The webpack build process is starting!!!');
    });
  }
}

module.exports = ConsoleLogOnBuildWebpackPlugin;

# 小结

正如上面所提到的webpack四大核心概览，当我们在了解到了entry,outpout,loader,plugin的相关含义，接下来我们就可以对相关概念进行深入的学习，比如了解更加详细的配置。

WARNING

要知道webpack的版本变化也是非常快的，学习webpack建议了解相关核心概念，知道其功能，解决的是什么问题，不需要死记硬背相关配置，因为可能在下一个版本配置就可能会发生变动，所以一项重要的技能就是查阅官方文档（英文官网)
