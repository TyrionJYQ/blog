# package.json

npm是nodejs包管理工具，可以通过npm来上传或者下载包，其中熟悉package.json文件的各个字段有助于我们对npm的理解。

# name

如果你想要在npm上发布开发的包，在package.json中最重要的两个字段是name和version字段。name指定包名，其他用户可以通过name来下载包。 version是版本号，在更改包内容再次上传包到npm时需要修改version.

WARNING

name相关规则

  • 大于或等于214个字符
  • 不能以.或者_开始
  • 不能出现大写字符

TIP

  • 不要使用和node核心模块相同的name
  • name中不要粗线js或者node
  • name可能会作为require的参数，所以尽可能简洁

# version

包的版本号

# description

字符串，能够使人们发现你发布的包，也作为npm包搜索的结果

# keywords

字符串数组，能够使人们发现你发布的包，也作为npm包搜索的结果

# homepage

项目主页url，一般为readme.md,例如

"homepage": "https://github.com/owner/project#readme"

# bugs

相关bug反馈途径，url或者邮箱，例如

{ "url" : "https://github.com/owner/project/issues"
, "email" : "project@hostname.com"
}

url和email可以指定一个或两个，当指定一个时，可以使用字符串代替对象。如果指定了url,会被npm bugs命令使用。

# files

npm包文件依赖 下面这些文件总会被包括，不管是否设置

  • package.json

  • README

  • CHANGES / CHANGELOG / HISTORY

  • LICENSE / LICENCE

  • NOTICE

  • The file in the “main” field 相反，以下文件通常会被忽略

  • .git

  • CVS

  • .svn

  • .hg

  • .lock-wscript

  • .wafpickle-N

  • .*.swp

  • .DS_Store

  • ._*

  • npm-debug.log

  • .npmrc

  • node_modules

  • config.gypi

  • .orig

  • package-lock.json (use shrinkwrap instead)

# main

main字段是项目入口文件的模块id，也就是说，如果你的包名为foo,用户下载了包并使用require('foo')来引入，这时main字段指定的文件导出的对象会作为require('foo')的返回值。 main字段的值是一个相对路径
